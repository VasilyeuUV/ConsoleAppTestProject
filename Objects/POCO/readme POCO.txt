POCO — Plain Old CLR Object

Основная цель POJO — показать, что домен может быть успешно смоделирован без сложностей, связанных со средой выполнения.
Среда выполнения вообще не должна иметь ничего общего с моделированием домена.

POCO выступает за использование максимально простых классов для объектов домена.

Классы POCO могут содержать логику.

Является надмножеством для DTO и Value Object, т.е. POCO может быть Value Object, Entity, DTO или любой другой класс, который вы можете создать, пока он не наследует сложность, случайную для вашего домена..


1. Сравнение:

								| DTO | ValueObject |  POCO  |
Содержит данные					| да  |     да      | да     |
Содержит логику					| нет |     да      | да/нет |
Имеет собственную идентичность	| нет |     нет     | да/нет |
