

Использование:
- где не требуется много инкапсуляции;
- для DTO (с init в частности);
- промежуточные классы данных (действуют как DTO между различными слоями системы) при:
	• загрузке данных в базу данных 
	• извлечении их из нее;
	• при выполнении некоторой предварительной обработки.
- для замены шаблона Fluent Interface (пример - Test Data Builder с with-функцией); 

Объекты значений DDD не входят в число тех, для которых можно использовать Record.
